# TimeDoctorAlert — Архитектура

## Общее описание

TimeDoctorAlert — кроссплатформенное консольное приложение (.NET 8+), работающее на Windows, macOS и Linux. Основная задача — мониторинг окон приложения Time Doctor через платформенные API и воспроизведение звукового оповещения при появлении нового окна. На Windows дополнительно поддерживается иконка в системном трее.

## Архитектурный паттерн

### Кроссплатформенная polling-утилита с платформенной абстракцией

Приложение использует паттерн Platform Abstraction — бизнес-логика отделена от платформенных реализаций через интерфейсы. Точка входа — `Program.cs` (top-level statements). Платформенные реализации выбираются в runtime через `PlatformFactory`.

## Интерфейсы платформенной абстракции

| Интерфейс | Назначение | Windows | macOS | Linux |
|---|---|---|---|---|
| `IWindowEnumerator` | Получение списка окон | Win32 EnumWindows | CGWindowListCopyWindowInfo | wmctrl |
| `IAudioPlayer` | Воспроизведение MP3 | NAudio WaveOutEvent | afplay (Process) | ffplay/mpg123 (Process) |
| `ITrayIcon` | Иконка в системном трее | WinForms NotifyIcon | No-op заглушка | No-op заглушка |

## Потоки данных

1. **Polling-цикл** (каждые 500мс): `WindowMonitorService.RunAsync` → `WindowTracker.Update` → `IWindowEnumerator.GetAllWindows()`
2. **Фильтрация**: Окна фильтруются по имени процесса ("Time Doctor" или "timedoctor2") и размеру (>550x100px)
3. **Обнаружение**: Сравнение текущего и предыдущего количества окон
4. **Оповещение**: При увеличении количества окон запускается MP3-воспроизведение в цикле (loop) через `IAudioPlayer`
5. **Остановка**: Звук останавливается при закрытии окна TD или по таймауту (1 минута)
6. **Логирование**: Все события отправляются через Serilog в консоль и на Seq-сервер

## Управление состоянием

- `WindowTracker._windows` — список текущих отслеживаемых окон (для отслеживания открытий/закрытий)
- `WindowMonitorService._windowsCount` — текущее количество отслеживаемых окон TD
- `CancellationTokenSource` — токены отмены для остановки мониторинга и воспроизведения

## Конфигурация

- **Seq URL** — `http://seq.n2home.keenetic.link` (embedded resource через `Assembly.GetManifestResourceStream`)
- **Фильтр окон** — определён в `WindowMonitorService` (процессы "Time Doctor", "timedoctor2", размер >550x100)
- **Интервал polling** — 500мс (жёстко закодирован)
- **Таймаут оповещения** — 1 минута (жёстко закодирован)
- **Воспроизведение** — MP3 проигрывается в цикле (loop) до отмены, в отличие от старого однократного воспроизведения

## Встроенные ресурсы

Ресурсы загружаются через `Assembly.GetManifestResourceStream` (класс `Resources.cs`), вместо старого подхода с `.resx` и сгенерированным `Resources.Designer.cs`.

## Тестирование

Тесты отсутствуют. Проект не содержит тестовых файлов или конфигураций.

## Безопасность

- URL Seq-сервера хранится как embedded resource — `http://seq.n2home.keenetic.link`
- На Windows используется P/Invoke для доступа к Win32 API — не требует повышенных привилегий
- На macOS используется CoreGraphics P/Invoke — не требует повышенных привилегий
- На Linux используется wmctrl — требует установки пакета
- Нет сетевых вызовов кроме отправки логов в Seq
